Hoisting In Javascript

Chắc hẳn nhiều bạn đã nghe thuật ngữ này rồi đúng không ? 🤔🤔🤔 Nó thường hay có trong những phỏng vấn interview về JavaScript. Mình mong rằng với những vấn đề mình nói dưới đây sẽ làm cho các bạn “Fully understand” về HOSTING nha. Let’s go 🏃‍♀️🏃‍♀️🏃‍♀️🏃‍♂️

Trước tiên, khi JavaScript chạy nó sẽ có 2️⃣ Pharse:

  • Pharse 1️⃣: Creation Pharse
    • Nó tạo từ khóa THIS và Global Object và 2 cái này = windows
    • Tiếp theo nó sẽ tạo ra HOISTING
  • Pharse 2️⃣: Execution Pharse
    • Run our code

Định nghĩa:

Hoisting là 1 hành động di chuyển của các biến (variable) hoặc function declarations lên đầu của environments xuyên suốt quá trình compile. Variable là 1 phần (partially) của hoisting and function declarations là hoisting

Nghe có vẻ rất khó hiểu đúng không nào 🤯🤯🤯🤯 Để mình giải thích cho các bạn bằng code nha.

Ở trên, dòng 5 mình khai báo variable mySong là 1 string: "giang"

Ở trên, dòng 9 mình khai báo 1 function declaration có tên là sing

Và khi các bạn run code này với các ngôn ngữ như C , C++ thì nó sẽ báo 1 lỗi ở dòng vì nó chưa được định nghĩa. Nhưng với JavaScript thì nó là 1 ra như thế này:

Câu hỏi đặt ra là tại sao lại là như thế này 🤔🤔🤔

Chính là vì JS engine chỉ định các bộ nhớ cho variables và function declarations xuyên suốtParse 1, trước khi nó sang Parse 2 (execute) nó . ➡ Đó là HOISTING
Vậy câu hỏi đặt ra tiếp theo là thằng JS engine nó làm gì ở bên sâu trong nó (underneath thehood) xuyên suốt Parse 1. ( Creation Phares) . JS engine sẽ nhìn qua code của chúng ta và ngay sau khi nó nhìn keyword (VAR and FUNCTION ) và nó “Ohh” nào hãy chỉ định memory bởi vì có keyword VAR and FUNCTION ➡ Nó cần memory và nó đặt variables mySong là undefined như mình mô tả ở dưới: Đó là lý do tại sao mình lại nói VARIABLE là 1 phần của hosting vì nó chỉ set undefined không phải là set value thẳng luôn . Với thằng function nó sẽ làm là nó sẽ chuyển thằng function mình đã khai báo lên trên thằng mình execute nó như hình dưới : Đó là lý do tại sao mình lại nói Function declartions are hoisted. Bất cứ khi nào mình gọi "execute" sing JS biết cái sing đó đang ở trong memory nào và sẽ return nó ra.
Làm 1 bài nhỏ các bạn đoán nhé . Bạn nghĩ sao về trường hợp này Như mình đã nói ở trên JS engine nó nhìn keyword VAR và Function thì nó sẽ hoisted. Nhưng khi JS nó nhìn tới dòng 6 nó nhìn là () ➡ nó k hoisted và nếu mình Run thì nó sẽ báo lỗi sing not define. ➡ Thêm nữa nhé : Nếu bạn đã học let , const rồi ES6 syntax thì nó cũng báo err luôn nha 💖💯
